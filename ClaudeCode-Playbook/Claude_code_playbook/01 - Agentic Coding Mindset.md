# 01 - Agentic Coding Mindset

> The foundational shift from writing code yourself to directing AI as a structured collaborator — not a random prompt machine.

---

## What It Is

Agentic coding means working with an AI coding tool in a disciplined, intentional way. You define goals clearly, provide the right context, and let the AI plan and execute multi-step tasks autonomously. You move from being a code writer to being a **technical director** — someone who understands the system and steers the AI toward the right outcome.

The contrast is **vibe coding**: prompting AI loosely, accepting whatever comes out, iterating blindly. Vibe coding works for throwaway experiments. Agentic coding is for real work.

> [!note] From the Video
> "Unlike vibe coding where you throw prompts and hope for the best, this teaches you to work with AI the right way: with structure, control, and production-level intent."

---

## Why It Matters

Vibe coding hits a ceiling fast. It breaks down when:

- The system needs to scale to real users
- The codebase grows beyond a single file
- Reliability, security, or correctness is non-negotiable
- You need to maintain and extend the code over time

Agentic coding removes that ceiling. Your role becomes **system-level understanding** — knowing what needs to be built and why — while the AI handles execution. The result is a 10x productivity multiplier without sacrificing quality.

> [!info] Added Context
> Think of it as the difference between being a craftsman who carves every piece by hand versus an architect who designs a building and directs skilled workers. The architect's output scales. The craftsman's doesn't.

---

## How to Use It

### The core mindset shift

- Stop thinking: *"What code do I write next?"*
- Start thinking: *"What do I need to get done, and what does the AI need to know to do it well?"*

### What this means in practice

- Give clear, scoped goals — not vague instructions
- Provide relevant context upfront (project structure, constraints, existing code)
- Review and steer the output rather than accepting it blindly
- Break large tasks into stages — plan first, execute second
- Use the AI's output as a starting point for your judgment, not the final word

> [!tip] Best Practice
> Treat Claude Code like a capable but context-blind new hire. The more clearly you brief it, the better the output. Ambiguous instructions produce ambiguous code.

---

## Best Practices

- Maintain **system-level awareness** at all times — know what the AI is building and why
- Never go fully hands-off on critical logic — review outputs before committing
- Use AI for execution-heavy tasks: boilerplate, refactoring, test writing, debugging loops
- Keep your own judgment active on architecture, security, and business logic
- Learn the tool's strengths — Claude Code excels at large-context understanding, architecture, and multi-step tasks

> [!tip] Best Practice
> The goal is not to reduce your thinking — it's to redirect it. You should be thinking harder about *what* to build and *why*, while the AI handles *how* to implement it.

---

## Common Mistakes

> [!warning] Common Mistake
> Using agentic tools with a vibe coding mindset — giving vague prompts and hoping for production-ready output. The tool is only as structured as your instructions.

> [!warning] Common Mistake
> Going fully hands-off on code review because "the AI wrote it." Agentic coding still requires you to understand and own the output.

- Skipping the planning stage and jumping straight to execution
- Treating every project as a vibe coding candidate — knowing when stakes are low vs. high matters
- Expecting the AI to infer context it was never given
